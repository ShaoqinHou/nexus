# i18n — Multi-Language Support

**Status:** In Progress
**Approach:** Single GLM translation service for everything — UI strings, menu content, user-generated text
**Quality gate:** Claude reviews GLM output, fixes issues

---

## Overview

One translation function powers everything. GLM (via z.ai Anthropic-compatible API) translates all content. Claude acts as QA reviewer, not translator.

```
translate(text, targetLocale, context) → translated text
```

Called at different times for different content types, but always the same pipe.

---

## Content Categories

### 1. Static UI Strings (~162 strings)
- Button labels, headings, placeholders, error messages, status labels
- **When translated:** Dev-time script extracts from JSX, sends to GLM in batches, writes locale JSON files
- **Storage:** `packages/web/src/locales/{locale}.json`
- **Maintenance:** `/translate` skill re-runs script, Claude reviews diff

### 2. Menu Content (DB-stored)
- Item names, descriptions, category names, modifier names, combo names, promotion text
- **When translated:** On save (API hook calls GLM), stored in `content_translations` table
- **Maintenance:** Auto on every save. Re-translatable via batch script.

### 3. Order Notes (time-sensitive)
- Customer special requests ("no peanuts, allergy")
- **When translated:** On order creation — auto-translated for kitchen staff safety
- **Storage:** `order_items.translatedNotes` or inline in content_translations
- **Why not on-demand:** Kitchen staff shouldn't have to click "Translate" for allergy info

### 4. Feedback Comments (not time-sensitive)
- Customer reviews after meal
- **When translated:** On-demand "Translate" button on staff dashboard
- **Storage:** Not stored — displayed temporarily

## Architecture

### Translation Service (`packages/api/src/lib/translate.ts`)

```typescript
interface TranslateOptions {
  text: string;
  targetLocale: string;
  context: string; // "menu item name", "UI button label", etc.
  sourceLocale?: string; // auto-detect if omitted
}

async function translate(options: TranslateOptions): Promise<string>
async function translateBatch(items: TranslateOptions[]): Promise<string[]>
```

Uses `@anthropic-ai/sdk` pointed at `https://api.z.ai/api/anthropic` with `ZAI_API_KEY`.
Model: `glm-5` (env override via `ZAI_MODEL`).

System prompt includes:
- "You are translating content for a restaurant ordering platform."
- "Preserve dish proper nouns (Pad Thai, Tiramisu, Fish & Chips)."
- "Use natural menu language, not literal word-by-word translation."
- "For CJK languages, provide phonetic rendering of foreign dish names."
- "Maintain any placeholder patterns like {count} or {name}."

### DB Table: `content_translations`

```sql
CREATE TABLE content_translations (
  id TEXT PRIMARY KEY,
  tenant_id TEXT NOT NULL REFERENCES tenants(id),
  entity_type TEXT NOT NULL,  -- 'menu_item', 'menu_category', 'modifier_group', etc.
  entity_id TEXT NOT NULL,
  locale TEXT NOT NULL,        -- 'zh', 'ja', 'ko', 'fr', etc.
  field TEXT NOT NULL,         -- 'name', 'description'
  value TEXT NOT NULL,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
CREATE UNIQUE INDEX idx_ct_lookup ON content_translations(tenant_id, entity_type, entity_id, locale, field);
```

### Frontend: Locale Context

```typescript
// packages/web/src/lib/i18n.ts
const LocaleContext = createContext<{ locale: string; t: (key: string) => string }>();

function useT(): (key: string) => string
// Reads locale JSON file, returns translation or falls back to English key
```

Customer language determined by:
1. URL param `?lang=zh`
2. localStorage preference
3. Browser `Accept-Language` header
4. Default: English

### Locale JSON Files

```
packages/web/src/locales/
  en.json  — source of truth (English strings as keys AND values)
  zh.json  — Chinese translations
  ja.json  — Japanese translations
  ko.json  — Korean translations
  fr.json  — French translations
```

Generated by dev-time script that:
1. Scans all .tsx files for `t('...')` calls
2. Extracts unique strings
3. Sends missing strings to GLM for each locale
4. Writes updated JSON files
5. Claude reviews the diff

### API: Locale Resolution

Customer API endpoints accept `Accept-Language` header or `?lang=` param.
Menu responses include translated content when locale != 'en'.

```typescript
// Middleware: resolveLocale
function resolveLocale(c: Context): string {
  return c.req.query('lang')
    || c.req.header('Accept-Language')?.split(',')[0]?.split('-')[0]
    || 'en';
}
```

### Tenant Settings

Add `supportedLocales: string[]` to tenant settings JSON.
Default: `['en']`. Restaurant owner enables locales via settings page.
Translation only runs for enabled locales.

## API Contracts

### Translation Endpoints (Staff)

```
POST /api/t/:tenantSlug/ordering/translate
Body: { text: string, targetLocale: string, context?: string }
Response: { data: { translation: string } }
Auth: Any staff
Purpose: On-demand translation (feedback, notes)

POST /api/t/:tenantSlug/ordering/translate/batch
Body: { items: [{ entityType, entityId, fields: { name, description }, targetLocale }] }
Response: { data: { translations: [...] } }
Auth: Owner/Manager
Purpose: Batch re-translate all menu content
```

### Localized Menu (Customer)

```
GET /api/order/:tenantSlug/ordering/menu?lang=zh
Response: Same shape as current, but name/description fields are translated
Fallback: Original language if no translation exists
```

## User Stories

### Customer
- **Given** I scan the QR code and my phone is set to Chinese
- **When** the menu loads
- **Then** I see category names, item names, and descriptions in Chinese
- **And** dish proper nouns (Pad Thai) show phonetic rendering + original name

### Kitchen Staff
- **Given** a Chinese customer places an order with note "不要花生"
- **When** the order appears on KDS
- **Then** the note shows auto-translated: "No peanuts" (below the original)

### Restaurant Owner
- **Given** I add a new menu item "Lamb Shank: Slow-braised with rosemary"
- **When** I click Save
- **Then** translations are auto-generated for all enabled locales
- **And** I can see/edit translations in the menu management page

### Developer (Claude)
- **Given** I added a new feature with 5 new UI strings
- **When** I run the `/translate` skill
- **Then** all locale files are updated with GLM translations
- **And** I review the diff for quality
